1. Какие существуют способы обработки различных ошибок?
а) с помощью алгоритма try{throw} - catch()
б) дебаггер, встроенный в компилятор
в) с помощью проверок if и вывода cout в каждом неоднозначном моменте
г) с помощью функции assert() 

2. В чём заключаются недостатки механизма кодов возврата?
Если много возможных мест с одиноковыми ошибками, то получается много кода.

3. Какими особенностями обладает механизм исключений?
Деструктор не кидает исключений;
Конструктор может кидать исключения, но должен сам выполнить очистки;
Блоки частных исключений выше общих, catch(...) самый последний;
Можно использовать механизм исключений для раскрутки стеков вызовов.

4. Для чего используется спецификатор и оператор noexcept?
Для того, чтобы отделять часть кода, не выбрасывающую исключений. Это проверяется ещё на этапе компиляции.

5.Как формулируются гарантии безопасности исключений?
1. Базовая (RAII):
	инварианты не нарушены
	нет утечек ресурсов
2. Строгая гарантия (банк)
	транзакционное поведение(нужно откатиться в случае чего)
3. Отсутствие исключений
